Contact Management System Automation 
Project Objective

This project was initially created to automate the Thinking Tester Contact Management Web Application.
However, due to continuous connectivity issues and the site becoming unavailable, full automation could not be executed.

To complete the automation objectives and retain all planned scenarios (Login, Data Entry, Edit, Delete, Session management, UI validation), the project was migrated to a publicly stable web application:

Automated Test Scenarios

Login Functionality
Valid/invalid login, empty fields, validation messages

Sign-Up / Registration
New user creation, invalid credentials, mandatory field checks

Add Items Module (Mapped from Add Contact)
Add products to cart, verify cart count, duplicate actions

Edit Functionality
Update cart quantity, verify changes

Delete Operation
Remove items from cart and validate cart status

Session & Navigation
Logout validation, session persistence, URL-based navigation

UI & Responsiveness
Element alignment, menu checks, visibility, layout behavior
